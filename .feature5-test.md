# Feature 5: Git-Diff Style Suggestions — Testing Guide

## Status: ✅ IMPLEMENTED

All three components are now in place:

### 1. System Prompt (server.js)
Kona now knows to output suggestions in this format:
```
<<<SUGGEST tab="script" section="Introduction">>>
---OLD---
Welcome to my channel!
---NEW---
Hey everyone! Welcome back to my channel!
<<<END_SUGGEST>>>
```

### 2. Frontend Parser (app.js)
- `processSuggestions()` — parses SUGGEST blocks and renders as diff UI
- `acceptSuggestion()` — applies the change to the correct editor
- `rejectSuggestion()` — dismisses the suggestion

### 3. CSS Styling (style.css)
- Green/red diff colors
- Accept/Reject buttons
- Accepted/Rejected states
- Warning state when text not found

## How to Test

1. Open Nora Writer: http://100.84.166.42:3000
2. Create or open a video project
3. Add some text to the Script editor
4. In the chat, ask Kona: "Can you suggest a better opening line for the introduction?"
5. Kona should respond with a SUGGEST block
6. You'll see:
   - A diff UI showing OLD (red) vs NEW (green)
   - Accept ✓ and Reject ✗ buttons
   - The tab and section context (e.g., "📝 script › Introduction")

7. Click Accept:
   - The text in the editor updates
   - Button changes to "✓ Applied" (green)
   - Diff fades to 60% opacity

8. Click Reject:
   - Nothing changes in the editor
   - Button changes to "✗ Dismissed" (red)
   - Diff fades to 40% opacity

## Example Prompts to Try

- "Suggest a more engaging opening for the intro section"
- "Make the description more SEO-friendly"
- "Improve the voiceover notes for section 2"
- "Rewrite this paragraph to be more concise: [paste text]"

## Technical Notes

- OLD text must match EXACTLY (including whitespace) for Accept to work
- If text isn't found, shows "⚠ Text not found in editor" warning
- Multiple SUGGEST blocks in one message work fine
- Tab parameter can be: script, description, voiceover, thumbnails
- Section parameter is optional but helps with context
