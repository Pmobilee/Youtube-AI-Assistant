## Implementation Summary

### ✅ Backend API Routes Added

1. **GET `/api/videos/:videoId/channels/:channelType/messages`**
   - Returns messages for a specific channel (script/description/thumbnail)
   - Validated channel types: `['script', 'description', 'thumbnail']`
   - Returns 400 for invalid channel types

2. **POST `/api/videos/:videoId/channels/:channelType/chat`**
   - Chat endpoint for specific channels
   - Uses channel-specific system prompts via `buildChannelSystemPrompt()`
   - Streams responses via SSE (Server-Sent Events)
   - Saves messages to `channel_messages` table with proper channel_type
   - Updates shared `video_memory` token count after each response

### ✅ Shared Memory Architecture

- **Table:** `video_memory` (per-video, shared across all channels)
- **Fields:** `video_id`, `summary`, `key_decisions`, `style_notes`, `token_count`, `updated_at`
- **Behavior:** All channels contribute to and read from the same memory
- **Token Counting:** Tracks tokens from ALL sources (old messages, channel messages, video content, memory summary)

### ✅ Channel-Specific System Prompts

**Function:** `buildChannelSystemPrompt(video, memory, globalMemory, channelType)`

**Behavior:**
1. Starts with base system prompt (includes shared memory)
2. Adds channel-specific focus instructions:
   - **script:** Focus on script content, structure, pacing, storytelling
   - **description:** Focus on YouTube description, SEO, metadata
   - **thumbnail:** Focus on thumbnail concepts, composition, visual impact
3. Adds cross-channel context (recent 3 messages from OTHER channels)

**Example prompt addition for script channel:**
```
## CURRENT FOCUS: Script Writing
You are in the Script chat. Focus on script content, structure, pacing, dialogue, and storytelling. When suggesting changes, use the <<<SUGGEST tab="script">>> format.

## Recent activity in description chat (for context):
user: Can you make the description more SEO-friendly?...
assistant: Here's an optimized version with better keywords...
```

## Database Schema

### channel_messages table
```sql
CREATE TABLE channel_messages (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  video_id INTEGER NOT NULL,
  channel_type TEXT NOT NULL DEFAULT 'script',
  role TEXT NOT NULL,
  content TEXT NOT NULL,
  image_url TEXT,
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  FOREIGN KEY (video_id) REFERENCES videos(id) ON DELETE CASCADE
);
```

### video_memory table (shared)
```sql
CREATE TABLE video_memory (
  video_id INTEGER PRIMARY KEY,
  summary TEXT DEFAULT '',
  key_decisions TEXT DEFAULT '[]',
  style_notes TEXT DEFAULT '[]',
  token_count INTEGER DEFAULT 0,
  updated_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  FOREIGN KEY (video_id) REFERENCES videos(id) ON DELETE CASCADE
);
```

## Key Implementation Details

### Channel Validation
```javascript
const validChannels = ['script', 'description', 'thumbnail'];
if (!validChannels.includes(channelType)) {
  return res.status(400).json({ error: 'Invalid channel type' });
}
```

### Cross-Channel Context
```javascript
const otherChannels = ['script', 'description', 'thumbnail'].filter(c => c !== channelType);
for (const ch of otherChannels) {
  const recentMsgs = db.prepare(
    'SELECT role, content FROM channel_messages WHERE video_id = ? AND channel_type = ? ORDER BY created_at DESC LIMIT 3'
  ).all(video.id, ch);
  if (recentMsgs.length > 0) {
    prompt += `\n\n## Recent activity in ${ch} chat (for context):\n`;
    recentMsgs.reverse().forEach(m => {
      const preview = m.content.substring(0, 200);
      prompt += `${m.role}: ${preview}${m.content.length > 200 ? '...' : ''}\n`;
    });
  }
}
```

### Streaming Response
- Uses Anthropic SDK with Claude Code OAuth
- Streams via SSE (text/event-stream)
- Saves complete response to database after streaming finishes
- Updates video timestamp and memory token count
